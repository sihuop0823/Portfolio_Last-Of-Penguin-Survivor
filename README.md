학교 기업 프로젝트 Last Of Penguin : Survivor에서 개발한 코드를 옮긴 리포지토리입니다

# 🐧 개발 내용 및 레포지토리 요약

#### 🌐 1. Server Connection UI (서버 연결 시스템)
onValidateInput을 활용해 입력 형식을 실시간 검증하고 공백을 자동 제거하여 데이터 무결성을 확보하였으며, 조건 충족 시에만 버튼이 활성화되는 구조로 불필요한 요청을 차단했습니다. 
또한 IPAddress.TryParse와 try-catch 기반의 예외 처리 및 타임아웃 핸들러를 도입하여 통신 오류 상황에서도 안정적인 접속 환경을 구축했습니다.

#

#### 🏗️ 2. UI Base System (UI 프레임워크 및 매니지먼트)
PanelBase 추상 클래스와 스택(Stack) 자료구조를 통해 UI 생명주기를 표준화하고 체계적인 화면 전환 및 히스토리 관리 기능을 구현하여 객체지향적인 유지보수 구조를 설계했습니다. 
UI 활성화 상태에 따른 커서 및 캐릭터 제어 자동화와 전역 락(Lock) 플래그를 통해 연출 중 오작동을 방지합니다.

#

#### 🐧 3. Character Selection UI (캐릭터 선택 시스템)
ScriptableObject 기반의 데이터 규격화와 싱글톤을 활용해 씬 전환 후에도 선택 정보를 안정적으로 보존하며, Render Texture와 전용 카메라를 통해 UI 내에 3D 모델을 투영화했습니다. 
또한 캐릭터별 시점 연출과 직관적인 스탯 시각화 시스템을 구축하여 사용자가 캐릭터 특성을 명확히 파악하고 선택할 수 있게 만들었습니다.

#

#### 💬 4. Chat & Alarm System (채팅 및 알림 시스템)
이벤트 기반 알림 및 메시지 처리
isNewMessage 이벤트를 구독하여 새로운 메시지 수신 시 UI 펄스 효과와 알림음을 독립적으로 처리하는 구조를 설계함으로써 시스템 간의 결합도를 낮췄습니다. 
또한 채팅창의 활성화 상태를 실시간으로 체크하여 불필요한 알림 발생을 방지했습니다

한글 IME 입력 최적화 및 동적 UI 시스템
유니티 엔진에서 흔히 발생하는 한글 자음과 모음의 분리 현상을 IME 조합 문자열 처리 로직을 통해 해결하였고, 또한 텍스트 길이에 따른 실시간 말풍선과 자동 스크롤 시스템을 도입하여 
많은 메시지가 발생하는 상황에서도 최적의 가독성을 유지하도록 설계했습니다.

#

#### 🤖 5. Buddy Dialog System
대화 데이터를 챕터별 SO로 구성해 코드 수정 없이 시나리오를 편집할 수 있는 구조를 설계했습니다. 
메인과 서브 레이어를 분리하고 스페이스바와 백스페이스 단축키를 지원해 대화 넘기기 및 이전 대화 확인 기능을 구현했습니다.
